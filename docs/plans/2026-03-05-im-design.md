# Slack-like IM 系统设计文档

## 概述

构建一个模仿 Slack 的即时通讯系统，用于验证 team-creator v6 多服务微服务架构下的 pipeline 能力。

**技术栈：** Rust（Axum）、PostgreSQL、Redis、React + TypeScript
**架构：** 3 个后端微服务 + 1 个前端，Docker Compose 本地部署
**目标并发：** 1000 并发 WebSocket 连接，单机部署

---

## 架构设计

### 服务拆分

```
demo-im/
├── api-service/          # :8080  REST API（持久化操作）
├── ws-service/           # :8081  WebSocket Hub（实时推送）
├── notification-service/ # :8082  通知处理（@mention、未读计数）
├── frontend/             # :3000  React SPA
├── docker-compose.yml
└── .pipeline/
```

### 服务交互图

```
Client (Browser/App)
  │
  ├── HTTP ──→ api-service:8080  (REST CRUD)
  │                │
  │                └── Publish → Redis Pub/Sub → ws-service (广播)
  │                └── Publish → Redis Pub/Sub → notification-service (通知处理)
  │
  └── WS ───→ ws-service:8081
                   │
                   └── Subscribe Redis → 推送给在线客户端

PostgreSQL ← api-service (写)、notification-service (读写)
Redis      ← 所有服务（在线状态、Pub/Sub、未读计数、rate limiting）
```

### 服务职责

| 服务 | 端口 | 职责 |
|------|------|------|
| api-service | 8080 | 认证、工作区/频道/消息 CRUD、文件上传、全文搜索、第三方 App、OAuth2 |
| ws-service | 8081 | WebSocket 连接管理、Redis 消息订阅广播、在线状态、typing 指示 |
| notification-service | 8082 | @mention 解析、未读计数、设备推送（FCM/APNS mock）、通知 API |
| frontend | 3000 | React SPA，WebSocket 长连接，频道/消息/通知 UI |

## 数据库设计

### 架构规则（强制）

> **禁止在 PostgreSQL 中使用 `ENUM` 类型。** 所有枚举字段存储为 `VARCHAR`，枚举值合法性校验在 Rust 业务逻辑层完成。违反此规则的 PR 必须被拒绝。

### PostgreSQL 表结构

```sql
-- 用户
CREATE TABLE users (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username     VARCHAR(50) UNIQUE NOT NULL,
    email        VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    avatar_url   VARCHAR(500),
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 工作区
CREATE TABLE workspaces (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name       VARCHAR(100) NOT NULL,
    slug       VARCHAR(50) UNIQUE NOT NULL,
    owner_id   UUID NOT NULL REFERENCES users(id),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE workspace_members (
    workspace_id UUID NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
    user_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role         VARCHAR(20) NOT NULL DEFAULT 'member',  -- 'owner'|'admin'|'member'
    joined_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (workspace_id, user_id)
);

-- 频道（含群聊/私信）
CREATE TABLE channels (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID REFERENCES workspaces(id) ON DELETE CASCADE,  -- NULL = 跨工作区私信
    name         VARCHAR(100),
    description  TEXT,
    channel_type VARCHAR(20) NOT NULL DEFAULT 'public',  -- 'public'|'private'|'direct'|'group_dm'
    created_by   UUID NOT NULL REFERENCES users(id),
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE channel_members (
    channel_id   UUID NOT NULL REFERENCES channels(id) ON DELETE CASCADE,
    user_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role         VARCHAR(20) NOT NULL DEFAULT 'member',  -- 'owner'|'admin'|'member'
    last_read_at TIMESTAMPTZ,
    notify_level VARCHAR(10) NOT NULL DEFAULT 'all',     -- 'all'|'mention'|'none'
    joined_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    invited_by   UUID REFERENCES users(id),
    PRIMARY KEY (channel_id, user_id)
);

-- 消息
CREATE TABLE messages (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    channel_id   UUID NOT NULL REFERENCES channels(id) ON DELETE CASCADE,
    sender_id    UUID NOT NULL REFERENCES users(id),
    content      TEXT,
    message_type VARCHAR(20) NOT NULL DEFAULT 'text',  -- 'text'|'file'|'app_card'|'system'
    thread_id    UUID REFERENCES messages(id),          -- NULL = 主消息，非 NULL = 线程回复
    app_card_id  UUID,                                  -- FK to app_cards
    edited_at    TIMESTAMPTZ,
    deleted_at   TIMESTAMPTZ,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_messages_channel_created ON messages(channel_id, created_at DESC);
CREATE INDEX idx_messages_thread ON messages(thread_id) WHERE thread_id IS NOT NULL;

-- 表情回应
CREATE TABLE reactions (
    message_id UUID NOT NULL REFERENCES messages(id) ON DELETE CASCADE,
    user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    emoji      VARCHAR(32) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (message_id, user_id, emoji)
);

-- 文件
CREATE TABLE files (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID REFERENCES workspaces(id),
    uploader_id  UUID NOT NULL REFERENCES users(id),
    filename     VARCHAR(255) NOT NULL,
    mimetype     VARCHAR(100) NOT NULL,
    size_bytes   BIGINT NOT NULL,
    storage_path VARCHAR(500) NOT NULL,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 第三方应用
CREATE TABLE apps (
    id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id   UUID NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
    name           VARCHAR(100) NOT NULL,
    app_type       VARCHAR(20) NOT NULL DEFAULT 'webhook',  -- 'webhook'|'bot'|'miniapp'
    webhook_url    VARCHAR(500),
    signing_secret VARCHAR(255) NOT NULL,
    scopes         TEXT NOT NULL DEFAULT '[]',  -- JSON array
    created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE app_cards (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    app_id     UUID NOT NULL REFERENCES apps(id) ON DELETE CASCADE,
    channel_id UUID NOT NULL REFERENCES channels(id) ON DELETE CASCADE,
    card_data  JSONB NOT NULL,   -- Slack Block Kit 兼容格式
    action_url VARCHAR(500),     -- 交互回调 URL
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- OAuth2（小程序/原生客户端）
CREATE TABLE api_clients (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id  UUID REFERENCES workspaces(id),
    name          VARCHAR(100) NOT NULL,
    client_type   VARCHAR(20) NOT NULL DEFAULT 'web',  -- 'web'|'mobile'|'miniapp'|'bot'
    client_id     VARCHAR(100) UNIQUE NOT NULL,
    client_secret VARCHAR(255) NOT NULL,
    redirect_uris TEXT NOT NULL DEFAULT '[]',           -- JSON array
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE oauth_tokens (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    client_id     UUID NOT NULL REFERENCES api_clients(id) ON DELETE CASCADE,
    user_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    access_token  VARCHAR(500) UNIQUE NOT NULL,
    refresh_token VARCHAR(500) UNIQUE NOT NULL,
    expires_at    TIMESTAMPTZ NOT NULL,
    scopes        TEXT NOT NULL DEFAULT '[]',
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 通知
CREATE TABLE notifications (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    notif_type  VARCHAR(20) NOT NULL,  -- 'mention'|'reply'|'reaction'|'app_action'
    message_id  UUID REFERENCES messages(id) ON DELETE CASCADE,
    is_read     BOOL NOT NULL DEFAULT FALSE,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_notifications_user_unread ON notifications(user_id, is_read) WHERE NOT is_read;

-- 设备推送（iOS/Android/Web）
CREATE TABLE devices (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token      VARCHAR(500) NOT NULL,
    platform   VARCHAR(20) NOT NULL,  -- 'fcm'|'apns'|'web_push'
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (user_id, token)
);
```

### Redis 数据结构

| Key 模式 | 类型 | 用途 | TTL |
|---------|------|------|-----|
| `online:{workspace_id}` | HSET `{user_id: timestamp}` | 在线状态 | 30s 滚动 |
| `typing:{channel_id}` | SET members | typing 指示 | 5s |
| `pubsub:channel:{channel_id}` | Pub/Sub | 消息实时广播 | — |
| `pubsub:notif:{user_id}` | Pub/Sub | 通知推送 | — |
| `unread:{user_id}:{channel_id}` | ZSET score=msg_timestamp | 未读计数 | — |
| `ratelimit:login:{ip}` | INCR | 登录频率限制 | 60s |
| `refresh:{user_id}:{jti}` | STRING | Refresh token | 7d |

## 深链接规范（跨客户端）

所有消息/频道响应体包含 `permalink` 字段：

```
im://{workspace_slug}/{channel_id}/{message_id}
```

各客户端自行注册协议处理器，API 统一输出此格式，不感知客户端类型。

## 安全策略

- **JWT**：HS256，access token 15 分钟，refresh token 7 天（Redis 存储 JTI）
- **WebSocket 鉴权**：握手时验证 query param `?token=`
- **文件上传**：类型白名单（image/*、video/*、pdf、office 文档）、50MB 限制、路径防穿越
- **App Webhook 验签**：HMAC-SHA256 `X-Signature` header，密钥为 `signing_secret`
- **Rate Limiting**：登录 5 次/分钟/IP，全局 API 100 次/分钟/用户（Redis INCR）
- **SQL 注入防护**：全程 SQLx 参数化查询，禁止字符串拼接 SQL

## 跨平台客户端兼容性

API 层面已完整支持后续原生客户端开发：

| 客户端 | 支持方式 |
|--------|---------|
| Web（当前） | REST + WebSocket，JWT Bearer |
| iOS / Android | REST + WebSocket + OAuth2 PKCE + 设备推送 API |
| Windows / macOS | REST + WebSocket（同 Web），深链接 `im://` |
| 小程序 | OAuth2 PKCE，`api_clients` 表管理 |
| 第三方 Bot | App Webhook + Bot token |

WebSocket 使用标准协议（WSS/443），MessagePack 二进制帧可选（`?encoding=msgpack`）；不采用原始 TCP Socket——在 1000 并发规模下帧头开销可忽略，且 WebSocket 可统一服务所有客户端类型。

## 验收标准

1. `GET /v1/health` 三个服务均返回 `{"status":"ok"}` HTTP 200
2. `POST /v1/auth/login` 返回 JWT access + refresh token
3. `POST /v1/channels/{id}/messages` 发送消息后，已连接的 WebSocket 客户端在 500ms 内收到 `new_message` 事件
4. `GET /v1/channels/{id}/messages` 返回分页消息历史（cursor-based）
5. @mention 消息触发 notification-service 写入 `notifications` 表，未读计数 +1
6. `POST /v1/apps/{id}/cards` 成功推送 app_card，WS 客户端收到 `new_message` 事件（type=app_card）
7. 单元测试 + 集成测试全部 PASS，覆盖率 ≥ 80%
8. `docker-compose up` 后所有服务健康检查通过
